bool *  consumed = nullptr

Parse the command line parameters passed to main and initialize all relevant args_t variables if the corresponding parameter is passed.

Returns
SUCCESS if sufficient parameters were successfully parsed or defaulted to execute the rest of the program.
Parameters
[in]argcThe same as argc from main. The number of elements in the argv array and the optional consumed array.
[in]argvArray of strings passed from main. Each is a token passed as a command line parameter.
[out]consumed. Optional. If you want to verify which command line parameters were recognized and consumed, allocate an pass an array of bools the same size as argv here. If a command line parameter (and possibly its argument) were consumed, their corresponding element in the consumed array is set to true. Note that elements are set to true even if an argument was invalid.

Returns
The same value that is written back through the arg_status argument.
Parameters
[in]argcNumber of elements in argv. Same as from main
[in]argvArray of tokenzied arguments passed to main
[in]long_parameterLong version of a command line parameter (such as "--width").
[in]short_parameterShort version of a command line parameter (such as "-w").
[out]arg_statusSee arg_status_t. This is always written back with the parsing status for the given function call.
[out]parameter_valueIf an argument is successfully parsed, the numeric value after that argument is written back. This pointer is not written back if parsing failed. For example, if argv[4] == "--width" and argv[5] == "42" then you could expect 42 to be written back via the parameter_value pointer.
